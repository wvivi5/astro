这种方法最稳，因为老棒子里那个镜像已经是被证明能跑的 ARM64 版本。

2. 然后用 WinSCP 连接【新棒子】，把这个 `migu_working.tar` 上传到新棒子的 `/opt`。

### 💡 为什么这个方法一定行？

* 避开了电脑 Docker 的 Bug：你电脑上的 Docker 引擎现在处于“混乱期”，怎么拉取都会报错。直接从 Linux 设备到 Linux 设备，不经过 Windows 转换。
* 架构百分之百正确：既然老棒子能跑，它导出的包绝对是 `arm64`。
