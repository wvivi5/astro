下面把今天关于 SSH 爆破 + fail2ban 的聊天重点先整理一下，然后给你一条“复制粘贴就装好”的一键命令（Ubuntu/CentOS 通用，规则按你最终要求配置）。



今天聊到的结论（简版）



1. 你的 VPS 公网 SSH（22）一直被扫描爆破是正常现象；日志里 成功登录来源基本是你自己的 IP 段。

2. 为了降低被撞库概率，我们决定用 fail2ban 自动封禁爆破 IP。

3. 规则从一开始的保守版（10m 内错 10 次封 30m）调整为你最终要的：

  • 3 分钟内错 10 次 → 封 6 小时

  • 白名单（永不封）：（并保留 127.0.0.1/8 ::1）

4. fail2ban 不需要重启 VPS 整机；改配置后重启 fail2ban 服务即可生效。

5. 要确保 系统重启后自动启动 fail2ban，必须执行 systemctl enable --now fail2ban。



───



一键安装 + 配置（Ubuntu/CentOS 都能用）



把下面整段在每台 VPS 上 SSH 登录后直接粘贴执行（会安装 fail2ban、写入规则、设置开机自启、立即启动）：



sudo bash -euxo pipefail <<'SH'

\# --- 1) install fail2ban ---

if command -v apt-get >/dev/null 2>&1; then

  export DEBIAN_FRONTEND=noninteractive

  apt-get update -y

  apt-get install -y fail2ban

elif command -v dnf >/dev/null 2>&1; then

  dnf install -y epel-release || true

  dnf install -y fail2ban

elif command -v yum >/dev/null 2>&1; then

  yum install -y epel-release || true

  yum install -y fail2ban

else

  echo "Unsupported distro: no apt/yum/dnf found" >&2

  exit 1

fi



\# --- 2) detect logpath (Ubuntu usually auth.log, CentOS usually secure) ---

LOGPATH=""

if \[ -f /var/log/auth.log ]; then

  LOGPATH="/var/log/auth.log"

elif \[ -f /var/log/secure ]; then

  LOGPATH="/var/log/secure"

else

\# fallback: let fail2ban decide, but still write something

  LOGPATH="%(sshd_log)s"

fi



\# --- 3) write jail config (your final rule + whitelist) ---

mkdir -p /etc/fail2ban/jail.d

cat >/etc/fail2ban/jail.d/sshd.local <<EOF

\[sshd]

enabled = true



\# 白名单（永不封）

ignoreip = 127.0.0.1/8 ::1 你的ip地址多ip空格隔开



\# 规则：3分钟内错10次，封6小时

findtime = 3m

maxretry = 10

bantime  = 6h



port = ssh

backend = auto

logpath = ${LOGPATH}

EOF



\# --- 4) enable autostart + start now ---

systemctl enable --now fail2ban

systemctl restart fail2ban



\# --- 5) show status ---

fail2ban-client ping

fail2ban-client status

fail2ban-client status sshd

systemctl is-enabled fail2ban

SH



验证/常用管理



sudo fail2ban-client status sshd

sudo fail2ban-client get sshd banip

sudo fail2ban-client set sshd unbanip 你的IP



───



两个小提醒（避免踩坑）



• 你填的 3 个白名单 IP 会“永不封”，确认它们确实是你自己的出口/跳板机 IP（你之前说是 Oracle 段，通常是你自己的云机出口）。

• 你现在还是 root + 密码登录的话，fail2ban 只能“降风险”，最稳还是后面改成密钥登录 + 禁止 root 直登。



你要不要我把脚本再加一个“安装后自动打印：当前封禁数/最近被封 IP/sshd 是否对公网监听”的检查输出？这样你每台执行完一眼就知道有没有生效。
