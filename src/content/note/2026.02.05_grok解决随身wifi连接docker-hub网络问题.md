核心文件：/etc/docker/daemon.json

{

  "registry-mirrors": [

\    "https://registry.ap-southeast-1.aliyuncs.com",

\    "https://hk.mirror.aliyuncs.com",

\    "https://docker.xuanyuan.me",

\    "https://docker.m.daocloud.io",

\    "https://mirror.ccs.tencentyun.com"

  ],

  "max-concurrent-downloads": 1,

  "proxies": {

\    "http-proxy": "socks5://127.0.0.1:1070",

\    "https-proxy": "socks5://127.0.0.1:1070",

\    "no-proxy": "localhost,127.0.0.1,::1"

  },

  "dns": \["8.8.8.8", "1.1.1.1"]

}

最关键的改动：



把 socks5h:// 改为 socks5://（去掉 h），解决了 lookup socks5h on 127.0.0.1:53: no such host 的错误。

优先使用阿里云新加坡节点（registry.ap-southeast-1.aliyuncs.com），对新加坡位置延迟最低。





完整成功过程回顾



最初问题

docker pull 一直卡在 net/http: TLS handshake timeout

即使设置了 export https_proxy=socks5h://127.0.0.1:1070 也无效

循环脚本报错（ash shell 的 $\[$n+1] 语法问题）



验证代理可用性curl -I https://www.google.com          # 成功返回 HTTP/2 200

curl https://registry-1.docker.io/v2/   # 返回 {"errors":\[{"code":"UNAUTHORIZED"...}]}→ 证明代理本身是通的，问题是 Docker daemon 没正确使用代理。

尝试 daemon.json 配置（早期失败）

最初用了 socks5h://，导致 lookup socks5h 错误

docker info 只显示默认 mirror（https://docker.zoey.netlib.re/），UCI 配置被忽略



关键转折：改成 socks5:// 并重启

修改 /etc/docker/daemon.json 中的 proxies 为 socks5://

然后强制重启：killall -9 dockerd

rm -f /var/run/docker.pid /var/run/docker.sock

/etc/init.d/dockerd restart || dockerd &检查确认：docker info | grep -i -A 10 -E "Mirror|Proxy"→ 显示所有 mirrors 和正确的 socks5://127.0.0.1:1070
